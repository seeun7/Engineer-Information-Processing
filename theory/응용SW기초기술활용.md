- 운영체제의 목적 
1. 처리 능력(Thoughput) 향상: 일정 시간 내에 시스템이 처리하는 일의 양 
2. 반환 시간(Turn Around Time) 단축: 시스템에 작업을 의뢰한 시간부터 처리가 완료될 때까지 걸린 시간
3. 사용 가능도(Availability) 향상: 시스템을 사용할 필요가 있을 때 즉시 사용 가능한 정도
4. 신뢰도(Reliability) 향상: 시스템이 주어진 문제를 정확하게 해결하는 정도

- 주기억장치 할당(연속 할당) 기법
1. 단일 분할 할당 기법: 주기억장치를 운영체제 영역과 사용자 영역으로 나누어 한순간에 한 명의 사용자만이 사용자 영역을 사용하는 기법. 경계 레지스터 사용(사용자 영역에 있는 사용자 프로그램이 운영체제 영역에 접근하지 못하도록 보호)
i. 오버레이 기법: 주기억장치보다 큰 사용자 프로그램을 실행하기 위한 기법
ii. 스와핑 기법: 하나의 프로그램 전체를 주기억장치에 할당해 사용하다가 필요에 따라 다른 프로그램과 교체하는 기법 
2. 다중 분할 할당 기법
i. 정적 할당 기법: 프로그램 할당 전 운영체제가 주기억장치의 사용자 영역을 여러개의 고정된 크기로 분할하고 준비상태 큐에서 준비중인 프로그램을 각 영역에 할당해 수행
ii. 동적 할당 기법: 주기억장치를 미리 분할해두지 않고 프로그램을 적재할 때 필요한 만큼의 크기로 영역을 분리하는 기법. 단편화를 줄이기 위해 사용   

- 주기억장치 할당(분산 할당) 기법
1. 페이징 기법: 가상기억장치에 보관되어 있는 프로그램과 주기억장치의 영역을 동일한 크기로 나눈 후 나눠진 프로그램을 동일하게 나눠진 주기억장치의 영역에 적재. 내부 단편화 발생 가능 
2. 세그멘테이션 기법: 가상기억장치에 보관되어 있는 프로그램을 다양한 크기의 논리적인 단위로 나눈 후 주기억장치에 적재시켜 실행. 외부 단편화 발생 가능

- 페이지 교체 알고리즘 
1. OPT: 앞으로 가장 오랫동안 사용하지 않을 페이지를 교체. 가장 효율적
2. FIFO: 가장 먼저 들어온 페이지부터 교체. 이해하기 쉽고 프로그래밍 및 설계 쉬움
3. LRU: 가장 오래 사용하지 않은 페이지 교체 
4. LFU: 사용 빈도가 가장 적은 페이지 교체 
5. NUR: 최근에 사용하지 않은 페이지 교체. LRU와 유사. 참조 비트와 변형 비트 사용 
6. SCR: 가장 오랫동안 주기억장치에 있는 페이지 중 자주 사용되는 페이지의 교체를 막기 위함. FIFO 단점 보완

- Locality: 프로세스가 실행되는 동안 주기억장치를 참조할 때 일부 페이지만 집중적으로 참조하는 성질이 있다는 이론  
- 워킹 셋: 프로세스가 일정 시간 동안 자주 참조하는 페이지들의 집합   
- 스래싱(Thrashing): 프로세스의 처리 시간보다 페이지 교체에 소요되는 시간이 더 많아지는 현상

- 프로세스 상태
1. 제출(submit): 작업을 처리하기 위해 사용자가 작업을 시스템에 제출한 상태
2. 접수(hold): 제출된 작업이 스풀 공간인 디스크의 할당 위치에 저장된 상태 
3. 준비(ready): 프로세스가 프로세서를 할당받기 위해 기다리고 있는 상태
4. 실행(run): 준비상태 큐에 있는 프로세스가 프로세서를 할당받아 실행되는 상태 
5. 대기(wait), 블록(block): 프로세스에 입출력 처리가 필요하면 현재 실행 중인 프로세스가 중단되고 처리가 완료될 때까지 대기 중인 상태
6. 종료(Terminated, Exit): 프로세스의 실행이 끝나고 할당이 해제된 상태   

- 프로세스 상태 전이
1. Dispatch: 준비 상태에서 대기중인 프로세스 중 하나가 프로세서를 할당받아 실행 상태로 전이되는 과정 
2. Wake UP: 입출력 작업이 완료되어 프로세스가 대기 상태에서 준비 상태로 전이되는 과정
3. Spooling: 입출력 장치의 공유 및 상대적으로 느린 입출력 장치의 처리 속도를 보완하고 다중 프로그래밍 시스템 성능 향상을 위해 입출력 데이터를 디스크에 저장해 한꺼번에 전달
4. Traffic Controller: 프로세스 상태에 대한 조사와 통보 담당  

- Thread: 시스템의 여러 자원을 할당받아 실행하는 프로그램의 단위 또는 프로세스 내에서의 작업 단위

- 주요 스케줄링 알고리즘
1. FCFS: 준비상태 큐에 도착한 순서대로 CPU 할당
2. SJF: 큐에 대기중인 프로세스 중 실행 시간이 짧은 프로세스 순서대로 할당
3. HRN: 우선순위를 계산해 높은 것부터 할당. 우선순위=(대기시간+서비스시간)/서비스시간

- OSI 참조 모델
1. 물리 계층
- 전송에 필요한 두 장치 간의 접속, 절단 등 기계, 전기, 기능, 절차적 특성에 대한 규칙 정의
- 물리적 전송 매체와 전송 신호 방식 정의
- 리피터, 허브
2. 데이터 링크 계층
- 2개의 인접한 개방 시스템 간 신뢰성 있고 효율적인 정보 전송을 위한 연결 설정, 유지, 종료 담당
- 송수신측 속도 차이 해결을 위한 흐름 제어 기능
- 프레임 동기화 기능, 오류 제어 기능, 순서 제어 기능
- 랜카드, 브리지, 스위치
3. 네트워크 계층
- 개방 시스템 간 네트워크 연결 관리, 데이터 교환 및 중계 기능. 네트워크 연결 설정, 유지, 해제 기능
- 경로 설정, 데이트 교환 및 중계, 트래픽 제어, 패킷 정보 전송 수행 
- 라우터
4. 전송 계층 
- 논리적 안정과 균일한 데이터 전송 서비스 제공해 end to end간의 투명한 데이터 전송 가능하게 함
- 7계층 중 하위 3계층, 상위 3계층의 인터페이스 담당 
- 게이트웨이
5. 세션 계층
- 송수신간 관련성 유지, 대화 제어. 대화 구성 및 동기 제어, 데이터 교환 관리
- 동기점: 오류가 있는 데이터의 회복을 위해 사용하는 것.
a. 소동기점: 하나의 대화 단위 내에서 데이터의 전달을 제어하는 역할. 수신 측으로부터 ACK 받지 않음
b. 대동기점: 전송 데이터의 처음과 끝에 사용해 데이터 단위를 대화 단위로 구성하는 역할을 함. 수신 측으로부터 반드시 ACK 받음
6. 표현 계층
- 응용/세션 계층으로부터 받은 데이터를 세션/응용 계층에 보내기 전 통신에 적당한 형태로 변환
- 서로 다른 데이터 표현 형태를 갖는 시스템 간 상호 접속을 위해 필요
- 코드 변환, 데이터 암호화, 데이터 압축, 구문 검색, 포멧 변환, 문맥 관리 등
7. 응용 계층
- 사용자가 OSI 환경에 접근할 수 있도록 서비스 제공
- 응용 프로세스 간의 정보 교환, 전자 사서함, 파일 전송, 가상 터미널 등의 서비스 제공

- 네트워크 관련 장비
1. 네트워크 인터페이스 카드(NIC): 컴퓨터과 컴퓨터/네트워크를 연결하는 장치. 정보 전송 시 케이블을 통해 전송될 수 있게 정보 형태를 변경. 이더넷 카드 or 네트워크 어댑터라고도 함
2. 허브(HUB): 가까운 거리의 컴퓨터 연결 장치로 각각의 회선을 통합해 관리. 신호 증폭(리피터) 역할 포함
a. 더미 허브: 네트워크에 흐르는 모든 데이터를 단순 연결 기능만 제공
b. 스위칭 허브: 네트워크에 흐르는 데이터의 유무 및 흐름 제어해 각 노드가 허브의 최대 대역폭을 사용할 수 있는 지능형 허브
3. 리피터(Repeater): 거리가 증가할 수록 감쇠하는 디지털 신호 전송을 위해 수신 신호를 재생 or 출력 전압 높여 전송하는 장치. 물리 계층에서 동작. 전송 거리 연장, 배선 자유도 높이기 위한 용도
4. 브리지(Bridge): LAN과 LAN을 연결하거나 LAN 안에서 컴퓨터 그룹을 연결하는 장치. 트래픽 병목 현상 줄일 수 있음. 네트워크를 분석적으로 구성할 수 있어 보안성 높일 수 있음. 서브넷 구성 시 전송 가능한 회선 수는 n(n-1)/2
5. 스위치(Switch): 브리지와 같이 LAN과 LAN을 연결해 더 큰 LAN을 만듦. 하드웨어 기반 처리로 전송 속도 빠름. 포트마다 서로 다른 전송 속도 지원 가능. 
6. 라우터(Router): 브리지와 같이 LAN, LAN 연결 기능에 데이터 전송 최적 경로 선택 기능 추가. 네트워크 계층까지의 프로토콜 구조가 다른 네트워크 간의 연결을 위한 프로토콜 변환 기능 
7. 게이트웨이: OSI 전 계층의 프로토콜 구조가 다른 네트워크를 연결. LAN에서 다른 네트워크에 데이터를 보내거나 다른 네트워크로부터 데이터를 받아들이는 출입구. 세션, 표현, 응용 계층 연결해 데이터 형식 변환, 주소 변환, 프로토콜 변환 등 수행

- 응용 계층 주요 프로토콜
1. FTP: 컴퓨터와 컴퓨터/인터넷 사이에서 파일을 주고받을 수 있게 하는 원격 파일 전송 프로토콜
2. SMTP: 전자 우편 교환 서비스
3. TELNET: 멀리 떨어져 있는 컴퓨터에 접속해 자신의 컴퓨터처럼 사용할 수 있게 해주는 서비스. 가상 터미널 기능 수행
4. SNMP: TCP/IP 네트워크 관리 프로토콜로 라우터, 허브 등 네트워크 기기의 네트워크 정보를 네트워크 관리 시스템에 보내는데 사용되는 표준 통신 규약 
5. DNS: 도메인 네임을 IP주소로 매핑
6. HTTP: www에서 HTML 문서 송수신하기 위한 표준 프로토콜

- 전송 계층 주요 프로토콜
1. TCP: 양방향 연결. 가상 회선 연결 형태의 서비스 제공. 스트림 위주의 패킷 단위 전달. 신뢰성 있음. 순서, 오류, 흐름 제어. 투명성 보장. 최대 100바이트
2. UDP: 비연결형 서비스. TCP에 비해 단순한 헤더 구조로 오버헤드가 적고 전송 속도 빠름(흐름, 순서 제어 없음). 실시간 전송에 유리하고 속도가 중요시되는 네트워크에서 사용
3. RTCP: RTP 패킷의 전송 품질 제어를 위한 프로토콜. 세션에 참여한 각 참여자들에게 주기적으로 제어 정보 전송. 데이터 전송 모니터링하고 최소한의 제어와 인증 기능만 제공. 하위 프로토콜은 데이터 패킷과 제어 패킷의 다중화 제공 

- 인터넷 계층의 주요 프로토콜
1. IP: 전송할 데이터에 주소 지정하고 경로 설정, 비연결형 데이터그램 방식, 신뢰성 보장 X
2. ICMP: IP와 조합해 통신 중 발생하는 오류 처리와 전송 경로 변경 등을 위한 제어 메시지 관리 역할
3. IGMP: 멀티캐스트 지원 호스트나 라우터 사이에서 멀티캐스트 그룹 유지를 위해 사용 
4. ARP: 호스트의 IP 주소를 호스트와 연결된 네트워크 접속 장치의 MAC Address로 바꿈
5. RARP: ARP와 반대로 물리적 주소를 IP주소로 변환 

- 네트워크 액세스 계층의 주요 프로토콜
1. 이더넷: CSMA/CD 방식의 LAN
2. IEEE 802: LAN을 위한 표준 프로토콜
3. HDLC: 비트 위주의 데이터 링크 제어 프로토콜
4. X.25: 패킷 교환망을 통한 DTE와 DCE 간의 인터페이스 제공 프로토콜
5. RS-232C: 공중 전화 교환망을 통한 DTE와 DCE 간의 인터페이스 제공 프로토콜 

- 네트워크 관련 신기술
1. IOT: 정보통신 기반 실세계와 가상세계의 다양한 사물들을 인터넷으로 연결해 진보된 서비스 제공
2. M2M: 무선 통신을 이용한 기계간의 통신
3. 모바일 컴퓨팅: 휴대용 기기로 이동하며 자유로이 네트워크에 접속해 업무를 처리할 수 있는 환경
4. 클라우드 컴퓨팅: 각종 컴퓨터 자원을 중앙 컴퓨터에 두고 인터넷 기능을 갖는 단말기로 언제 어디서나 인터넷을 통해 컴퓨터 작업 수행 가능한 가상화된 환경
5. 그리드 컴퓨팅: 지리적으로 분산되어 있는 컴퓨터를 초고속 인터넷 망으로 연결해 공유함으로써 하나의 고성능 컴퓨터처럼 활용하는 기술
6. 모바일 클라우드 컴퓨팅: 소비자와 소비자의 파트너가 클라우드 서비스를 이용해 모바일 기기로 클라우드 컴퓨팅 인프라를 구성해 여러 정보와 자원을 공유
7. 인터클라우드 컴퓨팅: 각기 다른 클라우드 서비스를 연동하거나 컴퓨팅 자원의 동적 할당이 가능하도록 여러 클라우드 서비스, 자원을 연결
8. 메시 네트워크: 차세대 이동 통신, 홈네트워킹, 공공 안전 등 특수 목적을 위한 새로운 방식의 네트워크 기술. 대규모 디바이스의 네트워크 생성에 최적화
9. 와이선(Wi-Sun): 스마트 그리드와 같은 장거리 무선 통신을 필요로 하는 IOT 서비스를 위한 저전력 장거리 통신 기술
10. NDN: 콘텐츠 자체의 정보와 라우터 기능만으로 데이터 전송을 수행하는 기술. IP를 대체할 새로운 인터넷 아키텍처로 떠오르는 중
11. SDN: 네트워크를 컴퓨터처럼 모델링해 여러 사용자가 각각의 소프트웨어로 네트워킹을 가상화하여 제어, 관리하는 네트워크 
12. NFC: 고주파를 이용한 근거리 무선 통신 기술. 아주 가까운 거리에서 양방향 통신을 지원하는 RFID 기술의 일종
13. UWB: 짧은 거리에서 많은 양의 디지털 데이터를 저전력으로 전송하기 위한 무선 디지털 펄스.땅속, 벽면 뒤로도 전송 가능 
14. 피코넷: 여러개의 독립된 통신 장치가 블루투스, UWB 통신 기술을 사용해 통신망을 형성하는 무선 네트워크 기술
15. WBAN: 웨어러블, 임플란트 형태의 센서나 기기를 무선으로 연결하는 개인 영역 네트워킹 기술 
16. GIS: 지리적 자료를 수집, 저장, 분석, 출력할 수 있는 컴퓨터 응용 시스템. 위성을 이용해 모든 사물의 위치 정보 제공해줌
17. USN: 각종 센서로 수집한 정보를 무선으로 수집할 수 있도록 구성한 네트워크. 필요한 모든 것에 RFID 태그 부착해 사물의 인식 정보와 주변 환경 정보까지 탐지해 정보 관리 
18. SON: 주변 상황에 맞춰 스스로 망을 구성. 통신망 커버리지 및 전송 용량 확장의 경제성 문제 해결하고 망의 운영과 관리 효율성 높임
19. 애드 혹 네트워크: 재난 현장과 같이 별도의 고정 유선망 구축할 수 없는 장소에서 모바일 호스트만을 이용해 구성한 네트워크
20. 네트워크 슬라이싱: 네트워크에서 하나의 물리적인 코어 네트워크 인프라를 독립된 다수의 가상 네트워크로 분리해 각각의 네트워크를 통한 고객 맞춤형 서비스 제공이 목적 
21. 저전력 블루투스 기술(BLE): 일반 블루투스와 동일한 주파수 내역을 사용하지만 대기상태일 땐 절전모드를 유지하는 기술
22. 지능형 초연결망: 스마트 시티, 스마트 스테이션 등 4차 산업혁명 시대를 맞아 새로운 변화에 따라 급격하게 증가하는 데이터 트래픽을 효과적으로 수용하기 위한 과학기술정보통신부 주관 사업
23. 파장 분할 다중화(WDM): 광섬유를 이용한 통신 기술. 파장이 서로 다른 복수의 신호를 보내 여러대의 단말기가 동시에 통신 회선을 사용할 수 있도록 하는 것.
24. 소프트웨어 정의 데이터 센터(SDDC): 데이터 센터의 모든 자원을 가상화해 인력의 개입 없이 소프트웨어 조작만으로 관리, 제어되는 데이터 센터. 컴퓨팅, 네트워킹, 스토리지, 관리 등을 모두 소프트웨어로 정의 
25. 개방형 링크드 데이터(LOD): 누구나 사용할 수 있게 웹상에 공개된 연계 데이터. 개별 URI로 식별하고 각 URI에 링크 정보를 부여해 상호 연결된 웹을 지향 

- 경로 제어 프로토콜
1. IGP(내부 게이트웨이 프로토콜): 하나의 자율 시스템 내의 라우팅에 사용되는 프로토콜
  a. RIP: 현재 가장 널리 사용되는 라우팅 프로토콜(=거리 벡터 라우팅 프로토콜). Bellman-Ford 알고리즘 사용됨. 최대 홉 수 15로 제한.
  b. OSPF: 대규모 네트워크에서 많이 사용. 다익스트라 알고리즘 사용. 노드 간 거리, 링크 상태 정보 실시간 반영해 최단 경로 라우팅 지원. 라우팅 정보에 변화가 생길 경우 변한 정보만 모든 라우터에 알림
2. EGP(외부 게이트웨이 프로토콜): 자율 시스템 간의 라우팅(= 게이트웨이 간의 라우팅에 사용)
3. BGP: EGP의 단점 보완. 초기 연결 시 라우팅 테이블 교환하고 이후엔 변화된 정보만을 교환

- 흐름 제어
1. Stop and Wait: 수신측의 ACK를 받은 후 다음 패킷 전송. 한번에 하나의 패킷만을 전송할 수 있음
2. Sliding Window: 확인 신호를 이용해 송신 데이터의 양을 조절. 확인 신호를 받지 않아도 미리 정해진 패킷 수만큼 전송. 효율 굿.. 긍정 ACK 수신 시 윈도우 크기 증가, 반대 시 감소

- 폭주 제어 
1. Slow Start: 윈도우 크기를 2배씩 증가시켜 갈수록 빨라짐. 전송 데이터 크기가 임계값에 도달하면 혼잡 회피 단계로 넘어감
2. Congestion Avoidance: 데이터 크기가 임계값에 도달 시 혼잡으로 간주하고 윈도우의 크기를 1씩 선형적으로 증가시켜 혼잡을 예방

- 네트워크 구축: 성형, 링형, 버스형, 계층형, 망형
- NAT(Network Address Translation): 1개의 정식 IP 주소에 대량의 가상 사설 IP 주소를 할당 및 연결하는 기능

- SW 관련 신기술
1. 인공지능: 인간의 두뇌와 같이 컴퓨터 스스로 인간지능적인 작업을 수행하는 시스템
2. 뉴럴링크: 일론머스크가 사람의 뇌와 컴퓨터를 결합하는 기술을 개발하기 위해 설립한 회사. Neural Lace(작은 전극을 뇌애 이식해 생각을 업로드, 다운로드하는 것) 개발을 목표
3. 딥러닝: 인간의 두뇌를 모델로 만들어진 인공신경망을 기반으로 하는 기계 학습 기술
4. 전문가 시스템: 의료 진단 등과 같이 특정 분야의 전문가가 수행하는 고도의 업무를 지원하기 위한 컴퓨터 응용 프로그램
5. 증강현실(AR): 실제 촬영한 화면에 가상의 정보를 부가해 보여주는 기술. 혼합 현실이라고도 함
6. 블록체인: P2P 네트워크를 이용해 온라인 금융 거래 정보를 온라인 네트워크 참여자의 디지털 장비에 분산 저장하는 기술
7. 분산 원장 기술(DLT): 중앙 관리자나 중앙 데이터 저장소가 존재하지 않고 P2P 망내의 참여자들에게 모든 거래 목록이 분산 저장되어 거래가 발생할 때마다 지속적으로 갱신되는 디지털 원장
8. 해시: 임의의 길이의 입력데이터, 메시지를 고정된 길이의 값이나 키로 변환
9. 양자 암호키 분해(QKD): 양자 통신을 위해 비밀키를 분배해 관리하는 기술
10. 프라이버시 강화 기술: 개인정보 침해 위협을 관리하기 위한 핵심 기술. 다양한 사용자 프라이버시 보호 기술 통칭
11. 공통 평가 기준: ISO 15408 표준으로 채택된 정보 보호 제품 평가 기준. 정보화 순기능 역할 보장을 위해 정보화 제품의 정보 보호 기능과 이에 대한 사용 환경 등급을 정한 기준
12. 개인정보 영향평가 제도(PIA): 개인 정보를 활용하는 정보시스템의 도입 및 변경 시 시스템의 구축, 운영이 기업의 고객 및 국민의 사생활에 미칠 영향에 대해 미리 조사, 분석, 평가하는 제도
13. 그레이웨어: 소프트웨어를 제공하는 입장에선 악의적이지 않다 주장할 수 있지만 사용자 입장에선 유용할 수도, 악의적일 수도 있는 애드웨어, 트랙웨어, 기타 악성 코드나 악성 공유웨어
14. 메시업: 웹에서 제공하는 정보 및 서비스를 이용해 새로운 소프트웨어나 서비스, 데이터베이스 등을 만드는 기술. 다수의 정보원이 제공하는 컨텐츠를 조합해 하나의 서비스로 제공하는 웹 사이트나 애플리케이션
15. 리치 인터넷 애플리케이션: 플래시 애플리케이션 기술과 웹 서버 애플리케이션 기술을 통합해 기존 HTML보다 역동적이고 인터렉티브한 웹 페이지를 제공하는 신개념의 플래시 웹 페이지 제작 기술
16. 시멘틱 웹: 컴퓨터가 사람을 대신해 정보를 읽고 이해, 가공해 새로운 정보를 만들어낼 수 있도록 이해하기 쉬운 의미를 가진 차세대 지능형 웹
17. 증발품(vaporware): 판매 계획, 배포 계획은 발표됐으나 실제 판매, 배포되지 않고 있는 소프트웨어
18. 오픈 그리드 서비스 아키텍처(OGSA): 애플리케이션 공유를 위한 웹 서비스를 그리드 상에서 제공하기 위해 만든 개방형 표준
19. 서비스 지향 아키텍처(SOA): 기업의 소프트웨어 인프라인 정보 시스템을 공유와 재사용이 가능한 서비스 단위나 컴포넌트 중심으로 구축하는 정보 기술 아키텍처
20. 서비스형 소프트웨어(SaaS): 소프트웨어의 여러 기능 중 사용자가 필요로 하는 서비스만 이용할 수 있도록 한 소프트웨어
21. 소프트웨어 애스크로: 소프트웨어 개발자의 지식재산권을 보호하고 사용자는 저렴한 비용으로 소프트웨어를 안정적으로 사용, 유지보수할 수 있도록 소스 프로그램과 기술 정보 등을 제3의 기관에 보관하는 것
22. 복잡 이벤트 처리(CEP): 실시간으로 발생하는 많은 사건들 중 의미가 있는 것만을 추출할 수 있도록 사건 발생 조건을 정의
23. 디지털 트윈: 현실 속 사물을 소프트웨어로 가상화한 모델

- HW 관련 기술
1. 고가용성: 긴 시간동안 안정적인 서비스 운영을 위해 장애 발생 시 즉시 다른 시스템으로 대체 가능한 환경을 구축하는 매커니즘
2. 3D 프린팅: 대상을 평면이 아니라 손으로 만질 수 있는 물체로 만들어내는 것
3. 4D 프린팅: 특정 조건이 갖춰지면 스스로 형태를 변화시키거나 제조되는 자가 조립 기술이 적용된 3D 프린팅하는 기술
4. RAID: 여러 개의 하드디스크로 배열을 구성해 파일을 구성하고 있는 데이터 블록들을 서로 다른 디스크에 분산 저장해 속도를 향상시킴
